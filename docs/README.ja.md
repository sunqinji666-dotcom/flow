## Flow とは

Flow は macOS 専用の軽量プロキシクライアントです。Xray、ノード検証、ローカルポート、システムプロキシ、ルーティング設定の複雑さを、落ち着いたインターフェースの裏側にまとめます。

メイン画面には接続状態、選択中のノード、そして大きな円形ボタンがあります。一度押すと接続し、もう一度押すと切断します。メニューバーからも接続、切断、ノード変更、ポート確認ができます。

Flow が大切にしているのは、次の三点です。

- 接続操作を簡単にする
- 現在の状態をわかりやすくする
- 公開ソースコードと秘密の接続情報を分離する

## 対象ユーザー

複雑なネットワーク管理画面を毎日操作せず、必要なときに素早く接続したい人向けです。通常は状態表示とメインボタンだけで利用できます。

多数のサブスクリプション、高度なルーティング編集、詳細ログ、サーバー管理が必要な場合は、NetFlow のような管理ツールの方が適しています。Flow はそれらを置き換えるものではありません。

## 接続の仕組み

```text
macOS アプリの通信
        ↓
Flow のシステムプロキシまたはローカルポート
        ↓
ローカル Xray コア
        ↓
自分のプライベートノード
        ↓
接続先ネットワーク
```

### 1. ノードを読み込む

以前に検証したノードを再利用するか、自分で設定したプライベート URL から一覧を取得します。公開リポジトリには作者の実用ノードは含まれません。

### 2. 実際の通信を検証する

サーバーのポートが開いているだけでは、プロキシが利用可能とは限りません。Flow は候補ノードごとに一時的なローカルプロキシを起動し、実際の通信が通るか確認します。

### 3. ローカルプロキシを起動する

検証後、正式な Xray プロセスを起動します。既定値は次のとおりです。

- SOCKS5：`127.0.0.1:10606`
- HTTP：`127.0.0.1:10607`

### 4. システムプロキシを適用する

システムプロキシを有効にすると、現在の macOS ネットワークサービスの HTTP、HTTPS、SOCKS 設定を Flow のローカルポートに向けます。切断時には設定を解除・復元します。

## 画面と基本概念

### 円形ボタン

接続と切断のメイン操作です。接続に成功すると、ボタン、状態テキスト、色、メニューバーアイコンがまとめて更新されます。

### ノード

通信の出口となるリモート接続先です。Flow は名前、プロトコル、トランスポート、検証結果を表示します。実際の認証情報は非公開です。

### システムプロキシモード

macOS のシステムプロキシに対応するアプリが自動的に Flow を利用します。独自のネットワーク機能を使うアプリは、この設定を無視する場合があります。

### ローカルポートモード

システムプロキシを無効にしても、Xray をローカルで動作させることができます。必要なアプリだけに SOCKS5 または HTTP ポートを設定します。

### ルーティングモード

- **ダイレクト**：対象通信をプロキシノードへ送らない
- **中国本土をバイパス**：一般的な中国本土向け通信を直接接続する
- **LAN をバイパス**：ローカルネットワークを直接接続する
- **グローバルプロキシ**：可能な限り対象通信をノード経由にする

### 通信量と接続時間

セッション、本日、累計の通信量と接続時間を表示します。動作確認のための目安であり、通信事業者レベルの課金記録ではありません。

## Flow と NetFlow の違い

| Flow | NetFlow |
| --- | --- |
| 日常利用向けの軽量接続クライアント | ノードとネットワークの総合管理ツール |
| 操作を絞り、状態を明確に表示 | サブスクリプション、ルーティング、ログなどを詳しく管理 |
| このリポジトリでは macOS 版のみを管理 | 別の目的を持つ独立プロジェクト |

このリポジトリには NetFlow、Android、Windows、Electron の実験コードは含まれていません。

## プライバシー境界

### 公開してよいもの

- SwiftUI の画面とアプリケーションロジック
- ビルドスクリプトと製品ドキュメント
- ダミー値を使ったノード構造
- 認証情報を含まない画像

### 非公開にするもの

- 実際のサーバー IP、ドメイン、接続先の組み合わせ
- UUID、Reality 公開鍵、shortId
- サブスクリプション URL とアクセストークン
- `.env`、SSH 秘密鍵、Cookie、Keychain データ

公開ソースのノードは意図的に利用できないダミー値です。リポジトリを複製しても、作者のプライベートネットワークにはアクセスできません。

## ダウンロードと設定

### ダウンロード

[GitHub Releases](https://github.com/sunqinji666-dotcom/flow/releases/latest) から最新版を取得できます。現在の公開ビルドは Apple Silicon（arm64）向けです。

パッケージが二つに分割されている場合は、Release ページのコマンドで結合し、SHA-256 を確認してください。

### プライベートノード URL を設定する

```bash
defaults write com.jacksun.flow FlowRemoteNodesURL "https://your-private-domain.example/flow/nodes.json"
```

ローカル設定を削除する場合：

```bash
defaults delete com.jacksun.flow FlowRemoteNodesURL
```

実際の URL やトークンを公開 Git リポジトリへ保存しないでください。

## ソースから実行・ビルド

### 実行

必要環境：macOS 14+、Swift 5.9+。

```bash
cd Sources/Flow
swift run
```

画面の確認はこれで可能です。実際に接続するには、Xray コアと有効な自分のノード設定が必要です。

### App をビルドする

```bash
FLOW_CORE_SRC="/your/local/path/Resources/Cores" \
FLOW_GEO_SRC="/your/local/path/routing-data" \
./Scripts/build_app.sh
```

出力先：

```text
build-output/Flow.app
```

## トラブルシューティング

### 画面は開くが接続できない

Xray 実行ファイルが存在するか確認し、その後プライベートノードが有効か確認してください。公開されているダミーノードでは接続できません。

### ポートには到達できるが検証に失敗する

ポート到達性とプロトコルの正しさは別です。UUID、Reality 公開鍵、SNI、shortId、プロトコル、セキュリティ、トランスポートの組み合わせを確認してください。

### 一部のアプリが直接接続する

macOS のシステムプロキシを無視するアプリがあります。そのアプリが対応している場合は、SOCKS5 または HTTP プロキシを個別に設定してください。

### 切断後もネットワークの状態がおかしい

Flow をもう一度開いてクリーンアップを実行してください。改善しない場合は、macOS のネットワーク設定で現在のサービスのプロキシ項目を確認してください。

### ノード JSON を公開してもよいですか

通常は推奨しません。完全なノード情報には接続認証情報が含まれるため、アクセス制御された場所に保存してください。
