## Flow 是什么

Flow 是一款只为 macOS 制作的轻量代理客户端。它把 Xray、节点检测、本地端口、系统代理和分流规则放在后台，把普通人真正需要判断的信息放在前面。

你打开 Flow 后，首先看到的是连接状态、当前节点和一个圆形按钮。点一下开始连接，再点一下停止。菜单栏也能完成连接、切换节点和查看端口。

Flow 的设计目标不是“功能最多”，而是这三件事：

- 连接动作足够简单
- 状态反馈足够明确
- 私有配置与公开源码严格分开

## 它适合谁

Flow 适合希望快速连接、又不想每天面对复杂设置的人。你可以把它当成一个简化的控制面板：平时只看状态和按钮，需要时再打开设置。

如果你要维护大量订阅、编辑复杂路由、查看详细日志或管理很多服务器，NetFlow 这类完整工具会更合适。Flow 不试图取代它们。

## 一次连接是怎么完成的

```text
Mac 应用网络
    ↓
Flow 的系统代理或本地端口
    ↓
本机 Xray 核心
    ↓
你自己的私有节点
    ↓
目标网络
```

### 1. 读取节点

Flow 优先使用已经验证并缓存的节点，也可以从你配置的私有地址读取节点列表。公开仓库没有附带作者的真实节点。

### 2. 做真实检测

单纯测试服务器端口只能说明“端口有回应”，不代表代理真的可用。Flow 会为候选节点临时启动本地代理，并用实际请求验证流量能否通过。

### 3. 启动本地代理

验证通过后，Flow 启动正式的 Xray 进程。默认提供：

- SOCKS5：`127.0.0.1:10606`
- HTTP：`127.0.0.1:10607`

### 4. 应用系统代理

如果你打开了“系统代理”，Flow 会把 macOS 当前网络服务的 HTTP、HTTPS 和 SOCKS 代理指向本地端口。断开时，Flow 会尝试恢复并清理这些设置。

## 界面与概念

### 大圆按钮

连接和断开的主操作。连接成功后，状态颜色、文字和菜单栏图标会同步变化。

### 节点

节点可以理解为“网络从哪里出去”。界面显示名称、协议、传输方式和检测结果。节点配置本身属于私有信息。

### 系统代理模式

开启后，遵守 macOS 系统代理的软件会自动经过 Flow。它方便，但并不保证所有应用都会遵守系统代理。

### 本地端口模式

关闭系统代理后，Xray 仍然可以运行。你可以在指定应用中手动填写 SOCKS5 或 HTTP 端口，只让这些应用经过 Flow。

### 分流策略

- **不代理**：直接连接，不经过代理节点
- **绕过大陆**：常见大陆网络直接连接，其他流量按规则处理
- **绕过局域网**：本地网络直接连接，其他流量经过代理
- **全局代理**：尽可能让所有匹配流量经过代理节点

### 流量与时长

Flow 显示本次、今日和累计流量，并记录连接时长。这些数据用于快速确认连接是否工作，不应当作运营商级计费依据。

## Flow 与 NetFlow 的区别

| Flow | NetFlow |
| --- | --- |
| 轻量、日常、一键连接 | 完整、专业、强调管理能力 |
| 只保留常用状态和设置 | 更适合订阅、节点、路由和详细数据管理 |
| 当前仓库只维护 macOS | 是另一个独立项目 |

当前 GitHub 仓库是 Flow，不包含 NetFlow 源码，也不包含 Android、Windows 或 Electron 实验版本。

## 隐私与公开边界

### 可以公开

- SwiftUI 界面与程序逻辑
- 构建脚本和产品说明
- 不含真实数据的节点结构示例
- 不包含凭据的图片

### 必须私有保存

- 真实服务器 IP、域名或端口组合
- UUID、Reality 公钥、shortId
- 订阅地址和访问 token
- `.env`、SSH 私钥、Cookie、Keychain 数据

公开源码中的节点是故意不能使用的占位符。下载源码后不会自动获得任何私人线路。

## 下载与配置

### 下载

前往 [GitHub Releases](https://github.com/sunqinji666-dotcom/flow/releases/latest) 下载最新版。当前公开包面向 Apple Silicon（arm64）。

如果安装包被拆成两个文件，请按照 Release 页面给出的命令合并，再核对 SHA-256。

### 配置私有节点地址

```bash
defaults write com.jacksun.flow FlowRemoteNodesURL "https://你的私有域名/flow/nodes.json"
```

删除这项本地设置：

```bash
defaults delete com.jacksun.flow FlowRemoteNodesURL
```

真实地址不要写入 Git 仓库。

## 从源码运行和打包

### 从源码运行

要求：macOS 14+、Swift 5.9+。

```bash
cd Sources/Flow
swift run
```

直接运行源码可以查看界面。要真正连接，还需要可用的 Xray core 和你自己的节点配置。

### 打包 App

```bash
FLOW_CORE_SRC="/你的本地路径/Resources/Cores" \
FLOW_GEO_SRC="/你的本地路径/路由数据" \
./Scripts/build_app.sh
```

输出位置：

```text
build-output/Flow.app
```

## 常见问题

### 界面能打开，但连接失败

先检查 Xray core 是否存在，再检查节点是否仍然有效。公开占位节点不能用于连接。

### 节点显示可达，但代理测试失败

端口可达不等于协议配置正确。检查 UUID、Reality 公钥、SNI、shortId、协议和传输方式是否匹配服务器。

### 打开系统代理后，部分软件仍然直连

有些应用不遵守 macOS 系统代理，或者使用自己的网络栈。这些应用可能需要单独填写 SOCKS5/HTTP 代理。

### 断开后网络异常

先退出并重新打开 Flow，让它执行清理；仍有问题时，在 macOS 网络设置中检查当前网络服务的代理选项。

### 能否直接公开我的节点 JSON

不建议。节点 JSON 通常包含完整连接凭据，应当放在有访问控制的私有服务中。
